# HERMES.md — shop-api

## 프로젝트 개요
React + Node로 만드는 쇼핑몰의 백엔드 API 서버. 주문/결제/상품 도메인을 다룬다.

## 에이전트 역할
기능 구현과 버그 수정을 돕는다. 새 엔드포인트 추가, 기존 로직 리팩터링, 테스트 작성이 주 업무다.

## 작업 규칙
- 코드를 바꾸기 전에 항상 먼저 계획을 한두 줄로 설명한다.
- 큰 변경은 작은 단위(커밋 가능한 단위)로 나눠서 진행한다.
- 작업이 끝나면 무엇을 왜 바꿨는지 불릿으로 요약한다.

## 하지 말 것
- 확인 없이 파일을 삭제하거나 덮어쓰지 않는다.
- 시키지 않은 리팩터링을 마음대로 하지 않는다.
- 비밀키·토큰을 코드에 하드코딩하지 않는다.

## 출력 형식
- 변경사항은 불릿 목록으로 요약한다.
- 실행/설치 명령은 복사 가능한 코드 블록으로 제시한다.

## 기술 스택
- TypeScript + Node(Express) + Prisma + PostgreSQL

## 자주 쓰는 명령
- `npm run dev` (개발 서버) · `npm test` (테스트) · `npm run lint`

## 디렉터리 구조
- `src/routes/` 라우트, `src/services/` 비즈니스 로직, `src/db/` Prisma 스키마, `tests/` 테스트

## 컨벤션
- 커밋은 Conventional Commits · 함수는 camelCase · 한 파일 400줄 이하

## 완료 기준
- 테스트 통과 + 린트 통과 + 변경 요약 제공 후 종료
