# Assembly Templates

질문 답변을 정리(Content Refinement)한 뒤 아래 템플릿에 채워 완성본을 만듭니다.
플레이스홀더(`{S1_이름}` 등)는 정리된 답변으로 치환하세요.

## SOUL.md

### 간단히
```markdown
# SOUL.md — {S1_이름}

## 정체성
- 이름: {S1_이름}
- 페르소나: {S2_페르소나}

## 말투와 태도
- 사용자 호칭: {S3_호칭}
- 말투 / 커뮤니케이션 스타일: {S4_말투} (직설성: {S4_직설성})
- 성격: {S5_성격}

## 커뮤니케이션 원칙
- 스타일상 피할 것: {S6_피할것}
- 불확실하거나 이견이 있을 때: {S7_대처}
```

### 자세히 (위에 더해)
```markdown
## 배경
{S8_배경}

## 취향
- 좋아하는 것: {S9_좋아함}
- 싫어하는 것: {S9_싫어함}

## 표현 습관
- 인사말: {S10_인사말} / 작별: {S10_작별}
- 특별한 습관: {S12_습관}
- 유머: {S14_유머}
- 감정 표현: {S13_감정}

## 관계와 목표
- 사용자와의 관계: {S11_관계}
- 궁극적인 목표: {S15_목표}
```

## HERMES.md

`CLAUDE.md` 스타일로, 에이전트가 읽기 쉬운 지침 문서로 조립합니다.

### 간단히
```markdown
# HERMES.md — {H1_에이전트}

## 에이전트 개요
{H1_목적}

## 에이전트 역할
{H2_역할}

## 작업 규칙
- {H3_규칙1}
- {H3_규칙2}
- {H3_규칙3}

## 하지 말 것
- {H4_금지1}
- {H4_금지2}

## 출력 형식
- {H5_형식}
```

### 자세히 (위에 더해)
```markdown
## 기술 스택
- {H6_스택}

## 자주 쓰는 명령
- {H7_명령}

## 디렉터리 구조
- {H8_구조}

## 컨벤션
- {H9_컨벤션}

## 완료 기준
- {H10_완료기준}
```

완성된 실제 샘플은 `examples/SOUL.example.md`, `examples/HERMES.example.md`를 참고하세요.
